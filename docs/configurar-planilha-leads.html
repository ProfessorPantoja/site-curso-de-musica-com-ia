📋 Ligar o formulário à sua planilha do Google

O formulário da landing (Nome + WhatsApp) já está pronto e funcionando. Falta só conectá-lo a uma planilha sua, para os contatos caírem automaticamente lá. Leva ~5 minutos e é grátis. Nenhum código no seu lado além de colar o trecho abaixo.

Por que assim e não Google Forms? O visitante preenche um formulário bonito, dentro do seu site (na sua identidade), e baixa o PDF na hora — isso converte melhor. Os contatos, por trás, caem numa planilha igualzinho ao Forms. O melhor dos dois mundos.

Parte 1 — Criar a planilha + o script

  1. 1 Acesse sheets.new para criar uma planilha nova. Dê um nome, ex.: Leads — Método Pantoja.
  2. 2 No menu, vá em Extensões → Apps Script. Vai abrir um editor de código numa nova aba.
  3. 3 Apague todo o conteúdo que estiver lá e cole exatamente este código: 
    // Recebe os cadastros do site e adiciona uma linha na planilha.
const ABA = "Leads";

function doPost(e) {
  try {
    const dados = JSON.parse(e.postData.contents);
    const planilha = SpreadsheetApp.getActiveSpreadsheet();
    let aba = planilha.getSheetByName(ABA);
    if (!aba) {
      aba = planilha.insertSheet(ABA);
      aba.appendRow(["Data/Hora", "Nome", "WhatsApp", "Origem"]);
    }
    aba.appendRow([
      new Date(),
      dados.nome || "",
      "'" + (dados.whatsapp || ""), // aspa = guarda o telefone como texto
      dados.origem || "landing"
    ]);
    return ContentService
      .createTextOutput(JSON.stringify({ ok: true }))
      .setMimeType(ContentService.MimeType.JSON);
  } catch (err) {
    return ContentService
      .createTextOutput(JSON.stringify({ ok: false, erro: String(err) }))
      .setMimeType(ContentService.MimeType.JSON);
  }
}
  4. 4 Clique no ícone de salvar (disquete) ou Ctrl+S.

Parte 2 — Publicar como "App da Web"

  1. 5 No canto superior direito, clique em Implantar → Nova implantação.
  2. 6 Em "Selecionar tipo" (ícone de engrenagem), escolha App da Web.
  3. 7 Configure assim:
    • Executar como: Eu mesmo (seu e-mail)
    • Quem pode acessar: Qualquer pessoa
  4. 8 Clique em Implantar. O Google vai pedir para autorizar — siga, escolha sua conta, e em "O Google não verificou este app" clique em Avançado → Acessar (nome do projeto). É seguro, é seu próprio script.
  5. 9 No fim, ele mostra a URL do app da Web (termina em /exec). Copie essa URL e me mande — ou guarde para a Parte 3.

Parte 3 — Conectar no site

É só dizer ao site qual é essa URL, através de uma variável de ambiente chamada LEAD_WEBHOOK_URL. Duas situações:

  1. Em produção (recomendado): no painel onde o site está hospedado (ex.: Vercel) → Settings → Environment Variables → adicione LEAD_WEBHOOK_URL com o valor da URL /exec. Salve e refaça o deploy.
  2. Para testar no seu computador: crie um arquivo .env.local na raiz do projeto com a linha: 
    LEAD_WEBHOOK_URL=https://script.google.com/macros/s/SEU_ID/exec
    e rode npm run dev.
Me manda a URL que eu ligo pra você. Se preferir, é só colar a URL aqui no chat — eu configuro a variável e confirmo que está caindo na planilha.

Bom saber

  • Enquanto a planilha não está conectada, o formulário continua funcionando e entregando o PDF — o lead só não é gravado ainda (fica registrado no log do servidor).
  • Se um dia mudar o código do script, é preciso fazer Implantar → Gerenciar implantações → Editar → Nova versão para valer.
  • O PDF entregue depois do cadastro fica em /public/iscas/metodo-pantoja-amostra.pdf. É só colocar o arquivo final aí com esse nome (ou ajustar o caminho em site.ts → leadMagnet.arquivoPdf).
